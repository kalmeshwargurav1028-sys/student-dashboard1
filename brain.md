## 📌 Project Summary

**Indus Portal** is a school Learning Management System (LMS) built with:
- **Backend:** Python + Flask (monolithic `app.py`)
- **Database:** MongoDB Atlas (PyMongo) + GridFS for binary files
- **Frontend:** Jinja2 templates + Tailwind CSS (CDN) + Vanilla JS
- **AI:** Google Gemini 2.5 Flash (`/api/analyze_student`, `/api/dashboard_ai`)
- **Hosting:** Vercel (serverless Python)
- **Notifications:** Twilio (SMS) + SendGrid (email)
- **Auth:** Flask sessions + Werkzeug + OTP via Office365 SMTP

MongoDB database name: `kalmeshwar`

## 👤 User Roles & Session Keys

Three roles exist in `session['role']`:

| Role | Login collection | Dashboard |
|---|---|---|
| `admin` | `admins` | `/admin_dashboard` |
| `teacher` | `users` | `/dashboard` |
| `student` | `student_users` (fallback: `students`) | `/student/<id>` |

Custom roles (e.g. `coordinator`) are stored in `users.custom_role` and map to permission entries in `db.role_permissions`.

**Session keys set on login:**
```
logged_in, role, user_id, username, email, photo_url
```

Session lifetime: **24 hours**.

## 🗃️ MongoDB Collections

| Collection | Key purpose |
|---|---|
| `students` | Academic profiles; app-level key is `id` (e.g. `IND001`) |
| `users` | Teacher accounts; `verified: true` required for login |
| `student_users` | Student auth credentials (separate from profiles) |
| `admins` | Admin accounts; default seeded on first login |
| `attendance` | One doc per day, `records: { student_id: status }` |
| `grades` | One doc per student-subject pair |

| `materials` | File metadata; actual file in GridFS |
| `timetable` | Schedule entries per class/day/subject |
| `daily_logs` | Teacher class notes per day |
| `announcements` | Broadcast messages with `audience` field |
| `notifications` | System bell notifications, soft-dismissed via `read_by[]` |
| `role_permissions` | Single doc (`_id: "global_config"`) — permission matrix |
| `settings` | Single doc — school config + API keys |
| `reports` | Cached analytics snapshot |
| `messages` | Parent-portal student messages |

> **GridFS** (`fs.files` / `fs.chunks`) stores photos, PDFs, and logos.

## 🔑 Key Conventions

### Student IDs
- Format: `IND` + zero-padded 3 digits → `IND001`, `IND002`, …
- Stored as `id` in the `students` collection (NOT `_id`)
- This is the **application-level key** used in all joins

### Attendance Values
- Stored as strings: `"Present"` / `"Absent"` / `"Late"`
- `recalculate_students_attendance()` runs **synchronously** on every save
  - Present = +1.0, Late = +0.5, Absent = +0.0
  - Result written back to `students[id].attendance` as a string

### File Storage
```python
# Upload
file_id = fs.put(file_object, filename=unique_name, content_type=mime_type)
file_url = url_for('get_file', file_id=str(file_id))

# Delete
fs.delete(ObjectId(file_id))
```
- URLs stored as `/file/<ObjectId>` in the collection document

### SMTP Config Priority
```
username → env MAIL_USERNAME  OR  db.settings.MAIL_USERNAME
password → env MAIL_PASSWORD  OR  db.settings.MAIL_PASSWORD
server   → db.settings.MAIL_SERVER  (NEVER from env — avoids stale Vercel cache)
port     → db.settings.MAIL_PORT    (NEVER from env)
```
> ⚠️ This is intentional. Do NOT refactor `_get_smtp_config()` to read server/port from env.

### OTP Flow
- 6-digit numeric code generated by `generate_otp()`
- Stored in session: `otp_code`, `otp_expiry` (+5 min), `otp_email`
- Verified by matching session value + checking `datetime.now() < otp_expiry`

### Gemini AI
- API key: `db.settings.GEMINI_API_KEY` OR `env.GEMINI_API_KEY`
- Model: `gemini-2.5-flash`
- **Stateless** — no conversation history stored server-side; chat state lives in browser JS only
- Asterisks (`*`) stripped from responses before rendering

## 🧩 Frontend Patterns

- **Base layout:** All pages `{% extends "base.html" %}` which provides sidebar, topbar, notification bell, flash messages
- **Sidebar color:** Emerald `#10b981`, collapses to icon-only via JS class swap
- **Role-gated nav:** Controlled by `session.role` + `role_permissions` context processor
- **JS:** All vanilla. Key patterns:
  - `fetch()` for AJAX (AI chat, notifications, announcements)
  - `setInterval(fn, 300000)` for 5-min notification polling
  - `data-*` attributes for live table filtering on dashboard roster
  - `confirm()` dialogs for all destructive actions

### Color Palette

| Admin accent | Red / Rose |
| Teacher accent | Blue / Indigo |
| Student accent | Purple / Fuchsia |
| Background | Gray-50 with dot-grid pattern |

## 🚨 Known Issues & Tech Debt

### Security (Must Fix Before Production)
1. **Plaintext passwords rendered in admin UI** — `{{ t.password }}` is visible in `admin_dashboard.html`
2. **Inconsistent password hashing** — Teacher signup stores plaintext; login accepts both via `check_password_hash` fallback
3. **`/dev/wipe_users` has NO auth check** — Wipes entire users collection; delete this route
4. **`SECRET_KEY` fallback** — Defaults to `'super_secret_key_change_in_production'` if env var missing
5. **No CSRF protection** — No Flask-WTF or custom tokens on any form
6. **API credentials in `db.settings`** — Gemini, Twilio, SMTP stored in MongoDB as plaintext

### Performance
- `recalculate_students_attendance()` is O(students × attendance_docs) and runs **synchronously** on every attendance save → move to background task at scale

### Architecture
- `app.py` is a single monolithic file. All logic, routes, and helpers coexist. No blueprints.
- MongoDB `maxPoolSize=1` set intentionally to avoid Atlas free-tier connection limits under serverless

## 🔧 Local Development

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Create .env file
MONGO_URI=mongodb://localhost:27017/
SECRET_KEY=your_secret_key
MAIL_USERNAME=your_office365_email
MAIL_PASSWORD=your_password
GEMINI_API_KEY=your_gemini_key   # optional
TWILIO_ACCOUNT_SID=...           # optional
TWILIO_AUTH_TOKEN=...            # optional
TWILIO_PHONE_NUMBER=...          # optional
SENDGRID_API_KEY=...             # optional
SENDGRID_FROM_EMAIL=...          # optional

# 3. Run
python app.py
# → http://localhost:5000
```

Default admin is **auto-seeded** on first login if `db.admins` is empty.

## 🚀 Deployment (Vercel)

- All traffic routed to `app.py` via `vercel.json`
- No persistent filesystem — all files go through GridFS
- Each request = new serverless instance (no shared in-memory state)
- Set all `.env` variables in the Vercel project dashboard
